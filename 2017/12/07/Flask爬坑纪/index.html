Flask 爬坑纪

发表于 2017-12-07 | 分类于 web框架 | |

前言

这几天因为老师项目需要，需要完成对医疗记录的录入以及管理，主要还是增删查改这几项，我选择了Flask框架来写。写的过程中，发现了很多问题，记录一下，方便以后的查阅。

为什么选择Flask

首要原因肯定是方便…,不用像SSH框架那样考虑各种jar包，并且Flask提供的丰富的扩展框架，也会方便一些特定功能的编写。比如Flask-wtf，对表单的快速渲染，Flaks-login，方便的管理用户在系统的认证状态。以及Sqlalchemy，让我们从数据库表格的复杂创建过程中脱离了出来，并且其中提供的对于关系的双向映射也非常有用。还有就是与之搭配使用的Migrate数据库迁徙框架，一是可以不用删除旧表，就可以动态改变表的结构。二是和git一样的作用，每修改一次数据库的模型，都会跟踪下这次的变化，方便恢复。
